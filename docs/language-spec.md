# WardLisp 言語仕様書

* 版: 0.2
* 対応実装バージョン: 0.2.0

---

## 1. 概要

WardLisp は、教育用 Lisp 学習ゲームのためにサーバサイドで安全に実行できる制限付き Lisp 方言である。

設計上の優先順位:

1. 安全性 -- ホストシステムへのアクセスを一切許さない
2. 資源制御 -- 計算量・メモリ・出力すべてに上限がある
3. 教育性 -- Lisp の本質（S 式、再帰、高階関数）を学べる
4. 決定性 -- 同一入力に対して常に同一結果を返す

Common Lisp 互換を目的としない。名前空間モデルは Scheme と同じ Lisp-1（関数と変数が同一の名前空間を共有する）を採用する。

## 2. 字句規則

### 2.1 文字セット

プログラムは ASCII テキストで記述する。

### 2.2 空白

以下の文字は空白として扱われ、トークンの区切りとなる:

* スペース (U+0020)
* タブ (U+0009)
* 改行 (U+000A)
* 復帰 (U+000D)

### 2.3 コメント

セミコロン `;` から行末までがコメントとなる。コメントは空白と同様に無視される。

```scheme
; これはコメント
(+ 1 2) ; 行末コメント
```

### 2.4 トークン

プログラムは以下のトークンから構成される:

| トークン | 説明 |
|----------|------|
| `(` `)` | リストの開始・終了 |
| `'` | クォート (quote の省略記法) |
| 整数リテラル | `0`, `42`, `-7`, `+3` |
| シンボル | `foo`, `null?`, `my-func` |
| `t` | 真値 |
| `nil` | 偽値 / 空リスト |

### 2.5 整数リテラル

整数リテラルは 10 進数で表記する。先頭に符号 (`+` または `-`) を付けることができる。

```
42      ;=> 42
-7      ;=> -7
+3      ;=> 3
0       ;=> 0
```

他の基数表記 (`#x`, `#o`, `#b`) はサポートしない。

### 2.6 シンボル

空白・括弧・クォート・セミコロン・`#` 以外の文字の連続がシンボルとなる。シンボルは読み取り時に小文字に正規化される。

```scheme
Foo     ; => foo と同一
NULL?   ; => null? と同一
my-func ; => my-func
```

#### 禁止される構文

* パッケージ接頭辞 (`:` を含むシンボル) は許可されない
* リーダマクロ (`#` で始まる構文) は許可されない

```scheme
cl:car      ; => parse-error
#'func      ; => parse-error
#(1 2 3)    ; => parse-error
```

## 3. データ型

WardLisp は以下のデータ型を持つ。

### 3.1 Integer (整数)

任意精度の整数。ただし絶対値に上限が設定される (既定: 2^64)。

```scheme
42
-100
0
```

### 3.2 Boolean (真偽値)

* `t` -- 真
* `nil` -- 偽

条件式において `nil` のみが偽として扱われる。`nil` 以外のすべての値（`0` を含む）は真として扱われる。

### 3.3 Nil

`nil` は偽値であると同時に空リストを表す。

### 3.4 Pair (ペア)

`cons` で作られる 2 要素の組。`car` で第 1 要素、`cdr` で第 2 要素を取得する。

```scheme
(cons 1 2)        ;=> (1 . 2)
(car (cons 1 2))  ;=> 1
(cdr (cons 1 2))  ;=> 2
```

### 3.5 List (リスト)

`nil` で終端されるペアの連鎖。

```scheme
(cons 1 (cons 2 (cons 3 nil)))  ;=> (1 2 3)
(list 1 2 3)                    ;=> (1 2 3)
'(1 2 3)                        ;=> (1 2 3)
```

### 3.6 Closure (クロージャ)

`lambda` または `define` で作られるユーザ定義関数。定義時の環境を閉包する。

```scheme
(lambda (x) (+ x 1))                ;=> #<closure>
(define (add1 x) (+ x 1))           ;=> #<closure add1>
```

### 3.7 Builtin (組み込み関数)

処理系が提供する組み込み関数。

```scheme
+        ;=> #<builtin +>
cons     ;=> #<builtin cons>
```

### 3.8 型の判定

| 関数 | 真を返す対象 |
|------|------------|
| `null?` | `nil` のみ |
| `atom?` | ペア以外のすべての値 (整数, `t`, `nil`, クロージャ, 組み込み関数) |

## 4. 評価規則

### 4.1 自己評価式

整数リテラル、`t`、`nil` はそれ自身に評価される。

```scheme
42    ;=> 42
t     ;=> t
nil   ;=> nil
```

### 4.2 シンボル参照

シンボルは現在の環境から値を探索する。見つからない場合は `name-error` となる。

```scheme
(let ((x 10)) x)   ;=> 10
y                   ;=> name-error: Undefined variable: y
```

### 4.3 リスト式の評価

リスト式 `(head args...)` は、`head` が特殊形式名の場合はその規則に従い、それ以外の場合は関数適用として評価される。

### 4.4 プログラムの評価

プログラムは 1 つ以上のトップレベル式から成る。各式は順に評価され、最後の式の値がプログラムの結果となる。

```scheme
(define x 10)
(define y 20)
(+ x y)          ;=> 30
```

## 5. 特殊形式

### 5.1 quote

```
(quote expr)
'expr
```

`expr` を評価せずにそのままデータとして返す。リストは内部表現のペア連鎖に変換される。

```scheme
'(1 2 3)      ;=> (1 2 3)
(quote foo)   ;=> foo (シンボルとしての文字列)
```

### 5.2 if

```
(if test then)
(if test then else)
```

`test` を評価し、結果が真 (`nil` 以外) なら `then` を、偽 (`nil`) なら `else` を評価して返す。`else` を省略した場合、偽のとき `nil` を返す。

```scheme
(if t 1 2)      ;=> 1
(if nil 1 2)    ;=> 2
(if nil 1)      ;=> nil
(if 0 1 2)      ;=> 1  (0 は真)
```

### 5.3 let (並列束縛)

```
(let ((name1 val1) (name2 val2) ...) body1 body2 ...)
```

並列束縛 (Scheme 方式)。すべての `val` を外側の環境で評価してから、結果を一括して `name` に束縛する。各 `val` の評価時に他の束縛は見えない。`body` を順に評価し、最後の式の値を返す。

```scheme
(let ((x 1) (y 2)) (+ x y))              ;=> 3
(let ((x 1) (y (+ x 1))) y)              ;=> name-error (x は束縛前)
```

### 5.3.1 let* (逐次束縛)

```
(let* ((name1 val1) (name2 val2) ...) body1 body2 ...)
```

逐次束縛。各束縛は前の束縛が見える環境で順に評価される。

```scheme
(let* ((x 1) (y (+ x 1))) y)             ;=> 2
```

> **注**: `let` と `let*` は異なるセマンティクスを持つ。依存関係のある束縛には `let*` を使用すること。

### 5.4 lambda

```
(lambda (params...) body1 body2 ...)
```

無名関数 (クロージャ) を作成する。複数の `body` がある場合は暗黙の `begin` として順に評価され、最後の値を返す。

```scheme
(lambda (x) (+ x 1))                     ;=> #<closure>
((lambda (x y) (+ x y)) 3 4)             ;=> 7
```

#### クロージャ

`lambda` は定義時の環境を捕捉する:

```scheme
(let ((n 10))
  (let ((add-n (lambda (x) (+ x n))))
    (add-n 5)))                           ;=> 15
```

### 5.5 define

#### 変数定義

```
(define name value)
```

`value` を評価し、現在の環境に `name` として束縛する。

```scheme
(define pi 3)
```

#### 関数定義

```
(define (name params...) body1 body2 ...)
```

以下の糖衣構文:

```scheme
(define name (lambda (params...) body1 body2 ...))
```

定義された関数は自身を再帰呼び出しできる:

```scheme
(define (fact n)
  (if (= n 0) 1
      (* n (fact (- n 1)))))
(fact 10)                                 ;=> 3628800
```

### 5.6 cond

```
(cond (test1 body1...) (test2 body2...) ... )
```

各節の `test` を順に評価し、最初に真となった節の `body` を暗黙の `begin` として順に評価し、最後の値を返す。すべての節が偽なら `nil` を返す。

`body` を省略した場合、`test` の評価結果自体を返す:

```scheme
(cond (nil 1) (42))                       ;=> 42
(cond (t (print 1) (print 2) 3))         ;=> 3 (出力: "1\n2\n")
```

### 5.7 and

```
(and expr1 expr2 ...)
```

左から順に評価し、最初に `nil` となった時点で `nil` を返す (短絡評価)。すべて真なら最後の値を返す。引数なしの `(and)` は `t` を返す。

```scheme
(and 1 2 3)       ;=> 3
(and 1 nil 3)     ;=> nil
(and)             ;=> t
```

### 5.8 or

```
(or expr1 expr2 ...)
```

左から順に評価し、最初に真 (`nil` 以外) となった値を返す (短絡評価)。すべて偽なら `nil` を返す。引数なしの `(or)` は `nil` を返す。

```scheme
(or nil nil 3)    ;=> 3
(or nil nil)      ;=> nil
(or)              ;=> nil
```

### 5.9 begin

```
(begin expr1 expr2 ...)
```

式を順に評価し、最後の式の値を返す。引数なしの `(begin)` は `nil` を返す。`lambda` や `define` の複数本体は暗黙の `begin` に変換される。

```scheme
(begin 1 2 3)     ;=> 3
(begin)           ;=> nil
```

### 5.10 apply

```
(apply func arg-list)
```

`func` と `arg-list` をそれぞれ評価し、`arg-list` の要素を引数として `func` に適用する。`arg-list` は正格リスト (`nil` で終端されたペア連鎖) でなければならない。

```scheme
(apply + '(1 2 3))                     ;=> 6
(apply (lambda (x y) (+ x y)) '(3 4)) ;=> 7
```

`func` はクロージャまたは組み込み関数でなければならない。`arg-list` が正格リストでない場合は `type-error` となる。引数個数が合わない場合は `arity-error` となる。

> **注**: `apply` は内部的には特殊形式として実装されているが、引数は通常の評価規則に従って評価される。

## 6. 関数適用

```
(func arg1 arg2 ...)
```

1. `func` を評価して関数値を得る
2. すべての `arg` を左から右に評価する
3. 関数を引数に適用する

関数値はクロージャまたは組み込み関数でなければならない。それ以外の値を関数位置に置くと `type-error` となる。

引数の個数が関数の仮引数と一致しない場合は `arity-error` となる。

### 6.1 Lisp-1 名前空間

関数と変数は同一の名前空間を共有する。`funcall` や `#'` は不要であり、サポートしない。

```scheme
(let ((f (lambda (x) (* x x))))
  (f 5))                                  ;=> 25

(define (apply-twice f x)
  (f (f x)))
(apply-twice (lambda (x) (+ x 1)) 0)     ;=> 2
```

## 7. 組み込み関数

### 7.1 算術演算

| 関数 | 引数 | 説明 | 例 |
|------|------|------|----|
| `+` | 0 個以上 | 加算。引数なしは `0` | `(+ 1 2 3)` => `6`, `(+)` => `0` |
| `-` | 1 個以上 | 1 引数なら符号反転、2 引数以上なら減算 | `(- 5)` => `-5`, `(- 10 3)` => `7` |
| `*` | 0 個以上 | 乗算。引数なしは `1` | `(* 2 3 4)` => `24`, `(*)` => `1` |
| `div` | 2 | 整数除算 (0 方向切り捨て) | `(div 7 2)` => `3` |
| `mod` | 2 | 剰余 | `(mod 7 2)` => `1` |

すべての算術関数は整数のみを受け付ける。整数以外を渡すと `type-error` となる。`div` および `mod` でゼロ除算を行うと `type-error` となる。

`div` は 0 方向への切り捨て (truncation toward zero) を行い、`mod` は `div` と整合する剰余 (truncation remainder) を返す。すなわち、常に `a = (div a b) * b + (mod a b)` が成り立つ。`mod` の結果は被除数と同符号になる:

```scheme
(div -7 2)   ;=> -3  (0 方向切り捨て)
(mod -7 2)   ;=> -1  (被除数と同符号)
(div 7 -2)   ;=> -3
(mod 7 -2)   ;=> 1
```

### 7.2 比較演算

| 関数 | 引数 | 説明 | 例 |
|------|------|------|----|
| `=` | 2 | 数値等値 | `(= 1 1)` => `t` |
| `<` | 2 | 小なり | `(< 1 2)` => `t` |
| `<=` | 2 | 小なりイコール | `(<= 2 2)` => `t` |
| `>` | 2 | 大なり | `(> 2 1)` => `t` |
| `>=` | 2 | 大なりイコール | `(>= 2 2)` => `t` |

比較関数は整数のみを受け付ける。結果は `t` または `nil` を返す。

### 7.3 リスト操作

| 関数 | 引数 | 説明 | 例 |
|------|------|------|----|
| `cons` | 2 | ペアを作成 | `(cons 1 2)` => `(1 . 2)` |
| `car` | 1 | ペアの第 1 要素 | `(car '(1 2))` => `1` |
| `cdr` | 1 | ペアの第 2 要素 | `(cdr '(1 2))` => `(2)` |
| `list` | 0 個以上 | 引数からリストを作成 | `(list 1 2 3)` => `(1 2 3)` |
| `null?` | 1 | `nil` なら `t` | `(null? '())` => `t` |
| `atom?` | 1 | ペアでなければ `t` | `(atom? 42)` => `t` |
| `length` | 1 | リストの長さ | `(length '(1 2 3))` => `3` |
| `append` | 0 個以上 | リスト結合 | `(append '(1 2) '(3 4))` => `(1 2 3 4)` |

* `car` と `cdr` は `nil` に対して `nil` を返す (エラーにならない)
* `car` と `cdr` はペアまたは `nil` 以外に対して `type-error` となる

#### append の挙動

`append` は 0 個以上のリストを結合する。最後の引数以外はコピーされ、最後の引数はそのまま結果の末尾として共有される。

```scheme
(append '(1 2) '(3 4))         ;=> (1 2 3 4)
(append '(1) '(2) '(3))        ;=> (1 2 3)
(append)                        ;=> nil
(append '(1 2))                 ;=> (1 2)
```

最後の引数以外がリストでない場合は `type-error` となる。各リストのコピー時に cons セルが消費される。

### 7.4 その他

| 関数 | 引数 | 説明 | 例 |
|------|------|------|----|
| `not` | 1 | 論理否定 | `(not nil)` => `t`, `(not 1)` => `nil` |
| `eq?` | 2 | 浅い等値 | `(eq? 1 1)` => `t` |
| `equal?` | 2 | 深い構造等値 | `(equal? '(1 2) '(1 2))` => `t` |
| `print` | 1 | 出力バッファに追記 | `(print 42)` => `42` (出力: "42\n") |

#### eq? の等値規則

| 型の組み合わせ | 比較方法 |
|--------------|---------|
| 整数同士 | 数値等値 |
| シンボル同士 | 文字列等値 |
| その他 | オブジェクト同一性 |

`eq?` はリスト構造の深い比較は行わない:

```scheme
(eq? '(1 2) '(1 2))    ;=> nil  (異なるペアオブジェクト)
(eq? 42 42)             ;=> t
```

#### print

`print` は値の文字列表現と改行を出力バッファに追記し、値そのものを返す。出力バッファの内容は実行メトリクスから取得できる。

#### equal? の等値規則

`equal?` は二つの値を再帰的に比較する深い構造等値判定を行う:

| 型の組み合わせ | 比較方法 |
|--------------|---------|
| 同一オブジェクト | 常に等値 (`eql` による同一性判定) |
| `nil` 同士 | 常に等値 |
| `t` 同士 | 常に等値 |
| 整数同士 | 数値等値 |
| シンボル同士 | 文字列等値 |
| ペア同士 | `car` と `cdr` を再帰的に `equal?` で比較 |
| その他 | 非等値 |

```scheme
(equal? '(1 2 3) '(1 2 3))             ;=> t
(equal? '(1 (2 3) 4) '(1 (2 3) 4))     ;=> t
(equal? '(1 2) '(1 3))                  ;=> nil
```

比較の深さは内部的に制限されている (10,000 レベル)。これを超えると `recursion-limit-exceeded` となる。

## 8. 環境モデル

### 8.1 レキシカルスコープ

WardLisp はレキシカルスコープを採用する。変数の参照は定義時の環境に基づく。

```scheme
(define (make-adder n)
  (lambda (x) (+ n x)))
(let ((add5 (make-adder 5)))
  (add5 10))                              ;=> 15
```

### 8.2 環境の構造

環境はフレームの連鎖で構成される。各フレームは名前と値の対応の集合である。変数参照は最も内側のフレームから外側に向かって探索される。

### 8.3 define のスコープ規則

トップレベルの `define` は現在の環境にフレームを追加する。同一スコープで後続の式から参照できる。

```scheme
(define x 10)
(define y (+ x 5))
y                       ;=> 15
```

## 9. 末尾呼び出し最適化

WardLisp は末尾位置にある関数呼び出しをトランポリン方式で最適化する。末尾再帰関数は反復回数に関係なく一定のスタック空間で実行される。

### 9.1 末尾位置

以下の位置にある式は末尾位置として扱われる:

| 特殊形式 | 末尾位置 |
|---------|---------|
| `if` | `then` 節、`else` 節 |
| `let` / `let*` | 本体の最後の式 |
| `begin` | 最後の式 |
| `cond` | 各節の結果式 |
| `and` | 最後の引数 |
| `or` | 最後の引数 |
| `lambda` | 本体の最後の式 (暗黙の `begin` 経由) |
| `define` | 本体の最後の式 (暗黙の `begin` 経由) |

### 9.2 末尾再帰の例

```scheme
;; 末尾再帰 — O(1) スタック空間
(define (sum-iter n acc)
  (if (= n 0) acc
      (sum-iter (- n 1) (+ acc n))))
(sum-iter 50000 0)                    ;=> 1250025000

;; 相互末尾再帰も最適化される
(define (even? n)
  (if (= n 0) t (odd? (- n 1))))
(define (odd? n)
  (if (= n 0) nil (even? (- n 1))))
(even? 10000)                         ;=> t
```

### 9.3 非末尾再帰

末尾位置にない再帰呼び出しは通常どおりスタックを消費する。`max-depth` 制限が適用される。

```scheme
;; 非末尾再帰 — (* n ...) が末尾位置を占めている
(define (fact n)
  (if (= n 0) 1
      (* n (fact (- n 1)))))
```

## 10. 値の表示

値は以下の形式で文字列化される:

| 型 | 表示 | 例 |
|----|------|----|
| Integer | 10 進表記 | `42`, `-7` |
| `t` | `t` | |
| `nil` | `nil` | |
| Symbol | シンボル名そのまま | `foo` |
| Proper list | `(a b c)` | `(1 2 3)` |
| Dotted pair | `(a . b)` | `(1 . 2)` |
| Closure | `#<closure>` または `#<closure name>` | `#<closure fact>` |
| Builtin | `#<builtin name>` | `#<builtin +>` |

## 11. 資源制御

WardLisp は実行中の資源消費を追跡し、制限を超えた場合に即座に実行を停止する。

### 11.1 ステップ制限 (fuel)

評価器は各ノードの評価でステップを消費する:

* 式の評価: 1 ステップ
* 関数適用: 追加 4 ステップ

残りステップが 0 以下になると `step-limit-exceeded` で停止する。

| パラメータ | 既定値 |
|-----------|--------|
| `fuel` | 10,000 |

### 11.2 再帰深さ制限

関数呼び出しのたびに深さカウンタが増加し、リターン時に減少する。上限を超えると `recursion-limit-exceeded` で停止する。

| パラメータ | 既定値 |
|-----------|--------|
| `max-depth` | 100 |

### 11.3 メモリ制限 (cons セル数)

`cons`、`list`、`quote` によるペア作成のたびにカウンタが増加する。上限を超えると `memory-limit-exceeded` で停止する。

| パラメータ | 既定値 |
|-----------|--------|
| `max-cons` | 10,000 |

### 11.4 整数サイズ制限

算術演算の結果の絶対値が上限を超えると `integer-limit-exceeded` で停止する。

| パラメータ | 既定値 |
|-----------|--------|
| `max-integer` | 2^64 |

### 11.5 出力制限

`print` による出力バッファのサイズが上限を超えると `output-limit-exceeded` で停止する。

| パラメータ | 既定値 |
|-----------|--------|
| `max-output` | 1,000 文字 |

---

## 12. エラー型

すべてのエラーは `wardlisp-error` を基底とし、以下の型に分類される:

| エラー型 | 発生条件 | 例 |
|---------|---------|-----|
| `parse-error` | 構文解析の失敗 | 閉じ括弧の不足、`#` の使用 |
| `name-error` | 未定義変数の参照 | `(+ x 1)` で `x` が未定義 |
| `type-error` | 型の不一致 | `(+ 1 t)`, `(car 42)` |
| `arity-error` | 引数個数の不一致 | `(+ 1 2)` に対する `(cons 1)` |
| `step-limit-exceeded` | ステップ上限超過 | 無限再帰 |
| `recursion-limit-exceeded` | 再帰深さ上限超過 | 深すぎる再帰 |
| `memory-limit-exceeded` | cons セル上限超過 | 巨大リスト生成 |
| `integer-limit-exceeded` | 整数サイズ上限超過 | `(expt 2 1000)` 相当の計算 |
| `output-limit-exceeded` | 出力文字数上限超過 | 大量の `print` |

## 13. ホスト API

### 13.1 evaluate 関数

```
(evaluate code &key fuel max-depth max-cons max-output max-integer timeout)
  => (values result metrics)    ; 正常終了
  => (values nil metrics)       ; エラー終了
```

| パラメータ | 型 | 既定値 | 説明 |
|-----------|-----|--------|------|
| `code` | string | (必須) | プログラムのソースコード |
| `fuel` | integer | 10,000 | 最大ステップ数 |
| `max-depth` | integer | 100 | 最大再帰深さ |
| `max-cons` | integer | 10,000 | 最大 cons セル数 |
| `max-output` | integer | 1,000 | 最大出力文字数 |
| `max-integer` | integer | 2^64 | 整数の絶対値上限 |
| `timeout` | integer | 5 | タイムアウト秒数 (予約) |

### 13.2 メトリクス

`evaluate` の第 2 返り値はプロパティリストで、以下の情報を含む:

| キー | 型 | 説明 |
|-----|-----|------|
| `:steps-used` | integer | 消費したステップ数 |
| `:max-depth-reached` | integer | 到達した最大再帰深さ |
| `:cons-allocated` | integer | 作成した cons セル数 |
| `:output` | string | `print` の出力内容 |
| `:fuel-remaining` | integer | 残りステップ数 |
| `:error-type` | symbol / nil | エラー型 (正常時は `nil`) |
| `:error-message` | string / nil | エラーメッセージ (正常時は `nil`) |

### 13.3 print-value 関数

```
(print-value value) => string
```

実行結果の値を表示用文字列に変換する。

## 14. 提供しない機能

以下の機能は安全性のため意図的にサポートしない:

* **マクロ** -- `defmacro`, `macrolet`
* **代入** -- `set!`, `setf`, `setq`
* **ループ** -- `loop`, `do`, `dotimes`, `dolist`
* **制御フロー** -- `tagbody`, `go`, `catch`, `throw`, `block`, `return`
* **副作用** -- `progn` (`begin` は特殊形式としてサポート)
* **自己評価** -- `eval`
* **入出力** -- ファイル、ネットワーク、OS コマンド
* **パッケージ** -- パッケージ操作、パッケージ接頭辞
* **読み取り時評価** -- `#.` その他のリーダマクロ
* **FFI** -- 外部関数呼び出し
* **スレッド** -- 並行処理
* **非決定性** -- 乱数生成、時刻取得、環境変数

---

## 15. 文法 (BNF)

```
program     ::= expr+

expr        ::= atom
              | '(' expr* ')'
              | "'" expr

atom        ::= integer
              | symbol
              | 't'
              | 'nil'

integer     ::= ['+' | '-'] digit+

symbol      ::= atom-char+    (* t, nil, integer 以外 *)

atom-char   ::= (* '(', ')', "'", ';', '#', 空白以外の文字 *)

digit       ::= '0' | '1' | ... | '9'

comment     ::= ';' (* 行末まで *)
```

## 16. 完全な例

### 階乗

```scheme
(define (fact n)
  (if (= n 0) 1
      (* n (fact (- n 1)))))
(fact 10)
;=> 3628800
```

### フィボナッチ数列

```scheme
(define (fib n)
  (cond ((= n 0) 0)
        ((= n 1) 1)
        (t (+ (fib (- n 1)) (fib (- n 2))))))
(fib 10)
;=> 55
```

### リスト反転

```scheme
(define (reverse lst)
  (define (rev-iter lst acc)
    (if (null? lst) acc
        (rev-iter (cdr lst) (cons (car lst) acc))))
  (rev-iter lst '()))
(reverse '(1 2 3 4 5))
;=> (5 4 3 2 1)
```

### 高階関数 (map)

```scheme
(define (my-map f lst)
  (if (null? lst) '()
      (cons (f (car lst)) (my-map f (cdr lst)))))
(my-map (lambda (x) (* x x)) '(1 2 3 4 5))
;=> (1 4 9 16 25)
```

### クロージャ

```scheme
(define (make-counter)
  (let ((count 0))
    (lambda ()
      (let ((current count))
        current))))
```

> **注**: 本言語は代入 (`set!`) を持たないため、カウンタの値を更新することはできない。上記は定義時の値を返すのみとなる。可変状態が必要なパターンは、代わりに再帰と累積変数で表現する。

### 末尾再帰による反復

```scheme
(define (sum-iter n acc)
  (if (= n 0) acc
      (sum-iter (- n 1) (+ acc n))))
(sum-iter 50000 0)
;=> 1250025000
```

### 挿入ソートと深い等値比較

```scheme
(define (insert x lst)
  (cond ((null? lst) (list x))
        ((<= x (car lst)) (cons x lst))
        (t (cons (car lst) (insert x (cdr lst))))))
(define (my-sort lst)
  (if (null? lst) nil
      (insert (car lst) (my-sort (cdr lst)))))
(equal? (my-sort '(3 1 4 1 5 9 2 6)) '(1 1 2 3 4 5 6 9))
;=> t
```

### apply によるリスト引数の適用

```scheme
(apply + '(1 2 3 4 5))
;=> 15

(define (my-sum lst)
  (apply + lst))
(my-sum '(10 20 30))
;=> 60
```
